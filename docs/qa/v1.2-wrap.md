## Operator-only work before beta ships

The master prompt §4 Bucket 8 explicitly calls these out as operator actions; this list is the concrete checklist:

1. **Import the repo into Vercel** (if not already) with framework = Next.js, install = `pnpm install`, build = `pnpm build`.
2. **Populate every env var** from `.env.example` for Production + Preview environments. The `deploy.md` runbook's "One-time: Vercel project" section walks through it — ensure `STRIPE_PRICE_ID_DIY` is among them (Bucket 1 F5 made it required, Bucket 8 deploy-runbook edit documents it).
3. **Fill Supabase + Stripe + Inngest + Resend** per their respective runbook sections.
4. **Deploy**. First deploy may fail on missing Stripe keys — that's expected; add them and redeploy.
5. **Run the 7-step post-deploy smoke** from `deploy.md` §"Post-deploy smoke". Every step must pass with zero console errors (§8 criteria #7, #8).
6. **Share the preview URL** as the beta.
