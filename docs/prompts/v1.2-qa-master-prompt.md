use the existing local checkout — either works as long as you end up on `main`:

```bash
git checkout main

```

If the tip is not `5572374` or later, `git pull origin main`. Do **not** create
a new branch; work directly against `main` with one PR per bucket.

Before reading further, run the clean-main bootstrap:

```bash
pnpm install --frozen-lockfile

pnpm typecheck
```

tiny PR, then proceed. `pnpm test:e2e` is opt-in (requires dev server); run it
at the end of each bucket, not as a precondition, and note that it is
`workflow_dispatch`-only in CI per ADR 0007.

### 2.3 Data model

`prisma/schema.prisma`. Key models:

- `User { role: CUSTOMER | ADMIN | ENGINEER | CPA }`
- `Property`
- `Study { tier: DIY | AI_REPORT | ENGINEER_REVIEWED; status: StudyStatus; ... }`
- `Document` (CLOSING_DISCLOSURE, IMPROVEMENT_RECEIPTS, PROPERTY_PHOTO, APPRAISAL, OTHER)
- `StudyEvent`, `AiAuditLog`, `Lead`
- `StudyShare { status: PENDING | ACCEPTED | REVOKED; token; invitedEmail; ... }`

**`StudyStatus` legal transitions** (DIY path is synchronous, not Inngest):

```
PENDING_PAYMENT → AWAITING_DOCUMENTS  (all tiers via webhook)
AWAITING_DOCUMENTS → PROCESSING       (Tier 1/2 after docs ready)
AWAITING_DOCUMENTS → DELIVERED        (DIY after form submit, synchronous)
PROCESSING → AI_COMPLETE → DELIVERED                                  (Tier 1)
PROCESSING → AI_COMPLETE → AWAITING_ENGINEER → ENGINEER_REVIEWED → DELIVERED  (Tier 2)
Any state → FAILED   (with failedReason)
Any state → REFUNDED
```

Any transition off this diagram is a bug. There is no centralized
`transitionStudy` helper today — the phase plan (§4) adds one as part of
Bucket 5.

## 5. Cross-cutting checklists

### 5.1 Security

- `SUPABASE_SERVICE_ROLE_KEY` imported only in files with `import "server-only"`
  at top. Grep confirms.
- `STRIPE_WEBHOOK_SECRET` verified before any DB write in
  `app/api/stripe/webhook/route.ts`. Malformed-signature test already landed (Day 60).
- Rate limits on: estimator, lead capture, magic-link send, checkout start,
  share invite, sample PDF. Each has a Vitest.
- `assertOwnership` invoked on every `Study`/`Document`/`Property` fetch in
  user-facing actions. Lint guard if feasible.
- Signed URL expiries: customer deliverable ≤ 7 days; admin inspector preview
  ≤ 1 hour.
- CSP header set in `next.config.ts` or middleware, allowlisting Sentry,
  PostHog, Supabase, Stripe, Inngest, Resend, Segra domains.
- No PII in Sentry breadcrumbs. Scrub at integration level.
- Auth role never downgraded on re-auth (`sync-user` must preserve existing
  role on upsert, not overwrite with `CUSTOMER`).
- Promo bypass (`FISHER_PROMO_CODE`): env-gated, non-falsy-required, constant-
  time compare if the comparison isn't already hardened.
- `classifyShareError`'s `wrong-account` path points at `/sign-in?signout=1`
  — verify that signs the user out before the sign-in form renders.

### 5.2 Data integrity

- Every Stripe webhook path is idempotent — guarded by
  `Study.stripeSessionId @unique` + test that delivers the same event twice
  and asserts one Study.
- Every Inngest step is idempotent — guarded by
  `AiAuditLog @@unique([operation, inputHash])` + test.
- Only legal `StudyStatus` transitions happen in the codebase. Add
  `lib/studies/transitions.ts` with a `transitionStudy(studyId, from, to, reason)`
  helper that enforces the graph in §2.3. Every status write goes through it.
  Unit test asserts illegal transitions throw.
- DIY synchronous path writes DELIVERED directly — ensure the transition
  helper knows `AWAITING_DOCUMENTS → DELIVERED` is legal for DIY tier only.
- Every orphan state has a recovery path: FAILED → rerun; stuck PROCESSING
  → force-fail or force-complete from the admin inspector.
- Every mutation writes a `StudyEvent` (kind string stable, payload
  JSON-serializable, `actorId` set for admin/customer actions, system for Inngest).

### 5.3 Error handling + observability

- Every route group has an `error.tsx` returning a friendly page +
  `Sentry.captureException`. Verify under `app/(marketing)`, `app/(auth)`,
  `app/(app)`, `app/(admin)`, `app/get-started`.
- Every server action returns a typed error result
  (`{ ok: true, ... } | { ok: false, code, message }`) and the client
  surfaces it in the UI, not only in logs.
- `console.log` purge — replace with `console.info/warn/error` and structured
  context object.
- PostHog captures at least: `estimator_submitted`, `lead_email_attached`,
  `sign_in_link_sent`, `checkout_started`, `study_created`,
  `documents_uploaded`, `study_processing_started`, `study_ai_complete`,
  `deliverable_downloaded`, `admin_rerun`, `admin_engineer_upload`,
  `delivery_email_sent`, `cpa_invited`, `cpa_accepted`. Any missing = bug.
- Sentry release tag + source maps (Vercel integration).

### 5.4 Accessibility (WCAG 2.2 AA minimum)

- `axe-core` clean on every page in §2.1 via `@axe-core/playwright`.
- Keyboard-only: estimator → sign-in → dashboard → intake → admin inspector.

## 7. Workflow

1. Post a TodoWrite plan for each bucket. Wait for operator "go".
2. One commit per fix (`qa(<surface>):` prefix). Smallest diff.
3. One PR per bucket. Title: `qa(<surface>): register + fixes`. Body:
   severity-ordered findings list with commit + test links.
4. Green CI on every push: `pnpm typecheck && pnpm lint && pnpm format:check
&& pnpm test && pnpm build`. Run `pnpm test:e2e` before opening the PR.
5. If a finding is > 30 min or touches another surface, STOP, file it, surface
   to operator before continuing.
6. Never edit the verbatim scope disclosure.
7. A bug without a regression test is not fixed.
